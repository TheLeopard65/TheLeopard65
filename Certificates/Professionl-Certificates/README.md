## HTB Certified Penetration Testing Specialist (CPTS)

The CPTS certification is an advanced 10-days (240 Hours) hands-on exam offered by Hack The Box (HTB) that validates a professional's ability to perform comprehensive penetration tests in real-world environments. It focuses on the entire penetration testing lifecycle, from reconnaissance to reporting, and emphasizes practical skills through immersive, simulated networks and machines. It includes penetration testing against Linux, Windows, Web Application, and Active Directory (AD) systems. By passing the HTB Certified Penetration Testing Specialist (CPTS), I have proven myself to have:

- Mastered the information gathering and reconnaissance techniques, including OSINT, passive scanning, active scanning, and search engine dorking.
- Exploited vulnerabilities in Windows & Linux environments, covering privilege escalation, pivoting, lateral movement, persistence, exploit chaining, port forwarding, & more.
- Conducted thorough web application security assessments, identifying, assessing, exploiting and reporting common flaws like SQL Injection, XSS, SSRF, LFI/RFI, IDOR, Command Injection, Insecure Design, and more.
- Utilized advanced penetration testing tools and frameworks (e.g., Metasploit-Framework, Burp Suite, PS-Empire, Cobalt-Strike) as well as gained hands on experience to manual pentesting as well.
- Developed custom exploit scripts and modified existing exploits to bypass modern defenses such has Windows Defender and Host-based Firewall, Active-Directory Defenses.
- Performed post-exploitation activities, including data collection and exfiltration, credential harvesting, covering tracks, installing and maintaining backdoor access.
- Gained hands on experience in writing professional penetration testing reports, documenting findings, security & business impact analysis, and providing remediation advice.
- Understood and applied various well-known penetration testing methodologies (e.g., PTES, OWASP) to structure assessments effectively and professionally.

## INE Certified Professional Penetration Tester (eCPPT)

The eCPPT certification is a 48-hours highly practical, professional-level certification from INE Security that tests a candidate's ability to conduct a full-scale penetration test against a multi-layered corporate network. It requires delivering a comprehensive penetration testing report, demonstrating both technical prowess and professional communication skills in a simulated real-world engagement. By passing the INE Certified Professional Penetration Tester (eCPPT), I have proven myself to have:

- Planned and executed advance penetration tests on complex network infrastructures, including multiple network segments, diverse OSs (Windows, Linux), AD and various network services.
- Performed deep-dive reconnaissance and vulnerability scanning using tools like Nmap, Nessus, and manual techniques to identify attack vectors across the entire target environment.
- Exploited a wide range of vulnerabilities (e.g., buffer overflows, web application flaws, system misconfigurations, weak credentials) to gain initial foothold and escalate privileges.
- Mastered pivoting and lateral movement techniques using tools like Proxychains, SSH Tunneling, and Metasploit to navigate through segmented networks and compromise additional systems.
- Conducted thorough post-exploitation activities, including password dumping with Mimikatz, keylogging, screen capture, and maintaining persistent access through backdoors and services.
- Developed a deep understanding of buffer overflow exploitation on both Windows and Linux platforms, including SEH overwrites, Egg Hunters, and custom shellcode generation.
- Produced a high-quality, executive-ready penetration test report, detailing the complete attack chain, technical findings, business impact analysis, and actionable remediation steps.
- Reinforced the importance of professional methodology, thorough documentation, and ethical conduct throughout every phase of a security engagement.

## INE Junior Penetration Tester (eJPT)

The eJPT certification is a foundational, entry-level certification from INE Security designed to validate core penetration testing skills through a 100% practical 48-hours exam. It focuses on essential techniques and tools used in real-world assessments, making it an ideal starting point for aspiring security professionals. By passing the INE Junior Penetration Tester (eJPT), I have proven myself to have:

- Performed network scanning and enumeration using Nmap to identify live hosts, open ports, running services, and operating system fingerprints.
- Executed manual and automated vulnerability assessments to discover common security weaknesses such as default credentials, misconfigurations, and unpatched services.
- Gained hands-on experience with the Metasploit Framework for exploiting vulnerabilities, handling sessions, and obtaining initial access to target systems.
- Conducted web application attacks, including directory traversal, parameter manipulation, and basic SQL injection to extract sensitive information.
- Understood the fundamentals of password attacks, including online brute-forcing with Hydra and offline hash cracking with John the Ripper.
- Navigated and managed multiple operating systems (Windows and Linux) in a penetration testing context, including file transfers and basic privilege escalation.
- Applied the core phases of penetration testing: information gathering, scanning and enumeration, exploitation, and post-exploitation fundamentals.
- Developed the ability to think like an attacker and systematically work through a controlled lab environment to achieve specific objectives.

## THM Junior Penetration Tester (PT1)

The THM Junior Penetration Tester certification (PT1) is a practical, beginner-friendly credential offered by TryHackMe, based on their comprehensive "Jr Penetration Tester" learning path. It validates foundational knowledge and hands-on skills required to start a career in penetration testing through gamified, real-world scenarios. By passing the THM Junior Penetration Tester (PT1), I have proven myself to have:

- Gained proficiency in using core penetration testing tools like Nmap for scanning, Gobuster for directory enumeration, and John the Ripper for hash cracking.
- Understood the principles of passive and active reconnaissance, including DNS enumeration, subdomain discovery, and OSINT techniques.
- Exploited basic web vulnerabilities, such as file upload flaws leading to remote code execution, Insecure Direct Object References (IDOR), and command injection.
- Learned to enumerate and exploit common network services, including SMB, FTP, SSH, and databases through practical, hands-on challenges.
- Practiced privilege escalation techniques on both Linux (e.g., SUID binaries, kernel exploits) and Windows (e.g., vulnerable services, token impersonation) systems.
- Developed foundational scripting skills in Python and Bash to automate simple security tasks such as port scanning and web requests.
- Acquired knowledge of common security concepts, such as hashing algorithms, encryption methods, and authentication mechanisms used in modern systems.
- Built a structured, repeatable approach to penetration testing by working through realistic, gamified rooms and capture-the-flag (CTF) style challenges.
